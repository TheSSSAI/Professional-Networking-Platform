# 1 Overview

## 1.1 Diagram Id

SEQ-UJ-005

## 1.2 Name

User Edits Profile Information

## 1.3 Description

A logged-in user navigates to their profile, enters edit mode, changes information (e.g., their headline or a work experience entry), and saves the changes.

## 1.4 Type

üîπ UserJourney

## 1.5 Purpose

To allow users to manage their professional identity on the platform, fulfilling SRS-001-F2 (1.2.1).

## 1.6 Complexity

Medium

## 1.7 Priority

üî¥ High

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-APP-CLIENT
- REPO-GW-API
- REPO-SVC-PROFILE
- REPO-EVENT-BUS

## 1.10 Key Interactions

- User clicks 'Edit Profile' on their own profile page.
- Client SPA enters an editable state and user modifies data in a form.
- User clicks 'Save'.
- Client sends the updated profile data to the API Gateway.
- API Gateway routes the request to the User Profile Service.
- User Profile Service validates the data, updates the records in PostgreSQL, and publishes a 'UserProfileUpdated' event.
- Client UI updates to show the saved information.

## 1.11 Triggers

- User saves changes to their profile.

## 1.12 Outcomes

- The user's profile is updated in the database.
- The changes are reflected in the search index (handled by SEQ-EP-001).

## 1.13 Business Rules

- Users can only edit their own profiles.
- All text fields must adhere to defined character limits and input validation (1.2.1).

## 1.14 Error Scenarios

- Submitted data fails validation (e.g., invalid date range for a job).
- A unique profile URL is already taken.

## 1.15 Integration Points

- Event Bus (for SEQ-EP-001).

## 2.2 Name

User Accesses and Interacts with the Help Center

## 2.3 Description

This sequence details the end-to-end user journey of accessing the static Help Center page. The flow is entirely client-side, initiated by a user clicking a link in the application footer. It covers navigation via the Next.js router, rendering of the page component, and user interaction with expandable FAQ sections (MUI Accordions), including state changes and re-rendering. This sequence fulfills requirement REQ-1-089.
