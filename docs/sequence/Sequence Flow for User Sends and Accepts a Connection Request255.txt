# 1 Overview

## 1.1 Diagram Id

SEQ-UJ-003

## 1.2 Name

User Sends and Accepts a Connection Request

## 1.3 Description

User A sends a connection request to User B. User B receives a notification and accepts the request, establishing a mutual connection.

## 1.4 Type

üîπ UserJourney

## 1.5 Purpose

To allow users to build their professional network, fulfilling SRS-001-F3 (1.3.1, 1.3.2).

## 1.6 Complexity

Medium

## 1.7 Priority

üö® Critical

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-APP-CLIENT
- REPO-GW-API
- REPO-SVC-CONNECTIONS
- REPO-EVENT-BUS
- REPO-SVC-NOTIFICATIONS

## 1.10 Key Interactions

- User A sends a connection request to User B from User B's profile.
- Connections Service creates a 'pending' connection record and publishes a 'ConnectionRequestSent' event.
- Notifications Service consumes the event and sends a real-time notification to User B.
- User B views their pending requests and clicks 'Accept'.
- Connections Service updates the record to 'accepted' and publishes 'ConnectionRequestAccepted'.
- Notifications Service notifies User A that their request was accepted.

## 1.11 Triggers

- User clicks 'Connect' on another user's profile.

## 1.12 Outcomes

- User A and User B are now first-degree connections.

## 1.13 Business Rules

- A connection is only established upon mutual acceptance (5.1.1).
- A user cannot send a request if one is already pending (5.1.1).

## 1.14 Error Scenarios

- The recipient user does not exist.
- The users are already connected.

*No items available*

# 2.0 Details

## 2.1 Diagram Id

SEQ-UJ-100

## 2.2 Name

User Accesses and Interacts with the Help Center

## 2.3 Description

This sequence details the end-to-end user journey for accessing the self-service Help Center. The process is initiated by a user click on a static link in the application footer and involves a purely client-side navigation and rendering flow within the Next.js application. The Help Center content is treated as static and does not require any backend API calls, emphasizing performance and accessibility.
