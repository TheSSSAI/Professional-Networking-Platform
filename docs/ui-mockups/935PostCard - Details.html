Description

The PostCard is a high-level organism component that serves as the primary container for displaying a single user-generated post in the news feed. It composes multiple molecular and atomic components to present the post's author, content (text, images, or link previews), engagement statistics, and action controls.

Design System Alignment

This component is built upon the atomic 'Card' component, inheriting its base styling. It integrates the UserBadge, ActionMenu, and ReactionPicker molecular components, and uses design system tokens for all spacing, typography, and color to ensure a cohesive look and feel.

Responsive Specifications

Mobile Adaptations

  • The component is fluid and adapts to a single-column layout, taking up the full width of its container.
  • The image grid maintains its structure but scales down with the container width.

Tablet Adaptations

  • No specific changes are needed; the fluid design scales appropriately.

Desktop Enhancements

  • Hover states on interactive elements like the user badge, action menu, and reaction picker provide richer feedback.
  • The layout has more available whitespace, making it easier to read.

Accessibility Features

Keyboard Support

All interactive elements (UserBadge link, ActionMenu, ReactionPicker, Comment/Share buttons) are focusable via Tab key and activatable via Enter/Space.

Screen Reader Support

The component is wrapped in an

tag for semantic structure. All interactive elements have descriptive aria-labels. User information and post content are read in a logical order.

Focus Management

A visible focus ring is applied to all focusable elements. The ActionMenu traps focus when open (requires JS, simulated with CSS).

Aria Attributes

  • aria-label for icon buttons
  • aria-haspopup/aria-expanded for ActionMenu

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
