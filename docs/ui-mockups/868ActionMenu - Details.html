Description

A compact menu component that provides a set of contextual actions within a dropdown. It is triggered by an icon button, typically a 'kebab' or 'three dots' icon, to save space in the UI.

Design System Alignment

Uses the design system's tokens for color, spacing, shadows, and border-radius to create a consistent look and feel. It's built upon the atomic 'Button', 'Icon', and a 'DropdownMenu' pattern based on the 'Card' component.

Responsive Specifications

Mobile Adaptations

  • The component has no specific responsive adaptations as its size is constant. The dropdown menu may be positioned to avoid viewport edges in a real JS implementation.

Tablet Adaptations

  • No specific adaptations needed.

Desktop Enhancements

  • Hover states on the trigger and menu items provide clear feedback.

Accessibility Features

Keyboard Support

The trigger is a standard button, focusable with Tab and activatable with Enter/Space. Menu items can be navigated with Tab. A JS implementation would add arrow key navigation.

Screen Reader Support

The trigger button must have an `aria-label` (e.g., 'More options for this post'). The dropdown has `role='menu'` and items have `role='menuitem'`, making the structure understandable to assistive technologies.

Focus Management

A CSS `:focus-within` pseudo-class is used to keep the menu open while focus is on any of its items. The focus outline is consistent with the design system.

Aria Attributes

  • aria-label
  • aria-haspopup='true'
  • aria-expanded='false/true'
  • role='menu'
  • role='menuitem'

Color Contrast

All text colors and backgrounds meet WCAG 2.1 AA contrast requirements.
