Description

A complex form component, typically presented in a modal, that allows users to create a new post with text, images, and link previews. It includes states for default input, submitting, and error feedback.

Design System Alignment

This organism is a composition of multiple atomic and molecular components, including Card, Avatar, TextArea, Button, FileUpload, and LinkPreviewCard. It heavily relies on the design system's tokens for layout, spacing, typography, and color to create a cohesive user experience.

Responsive Specifications

Mobile Adaptations

  • The modal becomes full-screen on mobile viewports (<768px), occupying 100% of the width and height for an immersive creation experience.
  • The layout remains a single vertical column, optimized for touch interaction.

Tablet Adaptations

  • The component uses the standard modal view, centered on the screen, as on desktop.

Desktop Enhancements

  • The component is presented as a centered modal, allowing the underlying page content to remain partially visible.
  • Hover states on action buttons provide clear interactive feedback.

Accessibility Features

Keyboard Support

The modal can be closed with the 'Escape' key. All interactive elements (close button, textarea, action buttons, post button) are focusable and operable via keyboard.

Screen Reader Support

The modal container uses `role="dialog"` and `aria-modal="true"`. It is labeled by the `h3` in the header via `aria-labelledby`. All buttons have descriptive `aria-label`s. Submitting and error states should be announced via ARIA live regions.

Focus Management

Focus is trapped within the modal when it is open. Upon opening, focus is programmatically moved to the `textarea`.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-label for icon buttons

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast ratios.
