Description

A molecular component that groups a form input (like TextInput or TextArea) with its corresponding Label and an optional validation Message. It ensures proper structure, spacing, and accessibility for a single form field.

Design System Alignment

This component utilizes the design system's spacing tokens for gaps between elements, typography tokens for the label and message, and semantic color tokens for error and success states.

Responsive Specifications

Mobile Adaptations

  • The default vertical stacking of label, input, and message is ideal for mobile viewports.

Tablet Adaptations

  • Maintains the default vertical stacking, which works well on tablet screen sizes.

Desktop Enhancements

  • The component's structure is flexible and can be placed within multi-column grid layouts on desktop screens.

Accessibility Features

Keyboard Support

The nested input is focusable via the Tab key.

Screen Reader Support

The `

Focus Management

The nested input has a clear `:focus-visible` state.

Aria Attributes

  • When in an error or success state, the message element's `id` should be linked to the input via `aria-describedby`.
  • In an error state, the input should have `aria-invalid="true"`.

Color Contrast

Label and message text colors for all states meet WCAG 2.1 AA contrast requirements against the background.
