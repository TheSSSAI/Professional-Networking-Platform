Component Name

Component Type

Description

A container for an administrator to review a reported piece of content (post or comment), including report metadata and moderation actions. It is designed to provide all necessary context for a moderation decision in a single view.

Design System Alignment

Built on the Card component, this organism composes atomic components like Avatar, Typography, and Button. It adheres to the design system tokens for color, spacing, shadows, and border radii.

Responsive Specifications

Mobile Adaptations

  • The metadata grid reflows to a single column layout.
  • The component itself is fully fluid and will take the width of its container.

Tablet Adaptations

  • The metadata grid adapts to show 2 or more columns depending on available width.

Desktop Enhancements

  • The metadata grid comfortably displays all items in a single row.

Accessibility Features

Keyboard Support

All interactive elements (links, buttons) are keyboard focusable and operable.

Screen Reader Support

Semantic HTML (

,
,
) is used to structure the component. Labels for metadata are explicitly provided. Loading and error states should be announced using ARIA live regions in a full implementation.

Focus Management

A clear, visible focus ring is applied to all interactive elements.

Aria Attributes

  • aria-live for dynamic state changes (loading/error)
  • aria-label for action buttons where context is needed

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
