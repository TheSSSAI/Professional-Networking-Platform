Description

A composite form control that combines a text input with an icon and an optional button, used for initiating user searches. It includes states for focus and loading, and has variants for global navigation and page-specific contexts.

Design System Alignment

This component utilizes the design system's tokens for colors (primary, neutrals), spacing (padding, gaps), typography, and border radius to ensure visual consistency with other components like TextInput and Button.

Responsive Specifications

Mobile Adaptations

  • The 'global_header' variant collapses to a single icon-only button to save space in the mobile navigation bar.
  • The 'page_specific' variant expands to occupy the full available width of its container.

Tablet Adaptations

  • The 'global_header' variant may still be collapsed on smaller tablets but becomes the full input on larger tablets.
  • The 'page_specific' variant maintains its standard desktop appearance.

Desktop Enhancements

  • Full input field is always visible for the 'global_header' variant.
  • Hover and focus states provide clear visual feedback.

Accessibility Features

Keyboard Support

The input field is focusable via the Tab key. The form can be submitted by pressing Enter.

Focus Management

A clear, high-contrast focus ring is applied to the entire component using `:focus-within` to indicate when the search bar is active.

Aria Attributes

  • role="search" on the parent form element.
  • aria-label on the input field, provided by a visually hidden label.

Color Contrast

Placeholder text and icon colors meet WCAG AA contrast ratios against the background.
