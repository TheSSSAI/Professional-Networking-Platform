Description

A modal dialog that interrupts the user's workflow to ask for confirmation before proceeding with an action. It includes variants for standard and destructive actions, ensuring the user understands the consequence of their choice.

Design System Alignment

This component is built upon the atomic 'Modal' and 'Button' components. It uses design system tokens for typography (headings, body), colors (primary, semantic error, neutrals), spacing, shadows, and border-radius to ensure a consistent and integrated user experience.

Responsive Specifications

Mobile Adaptations

  • The dialog itself adapts to the viewport width with padding.
  • The action buttons stack vertically to be more touch-friendly and prevent accidental clicks.

Tablet Adaptations

  • Maintains the desktop layout with side-by-side action buttons.

Desktop Enhancements

  • Centered modal with a backdrop overlay, providing clear focus on the confirmation task.

Accessibility Features

Keyboard Support

In a JS implementation, focus must be trapped within the modal. All buttons are focusable and activatable via keyboard. The modal can be dismissed with the 'Escape' key.

Screen Reader Support

The dialog uses `role="dialog"`, `aria-modal="true"`, `aria-labelledby` to point to the title, and `aria-describedby` to point to the description. This provides a complete and accessible context to screen reader users.

Focus Management

Upon opening, focus should be moved to the first focusable element inside the dialog (typically the 'Cancel' or 'Confirm' button). Upon closing, focus should return to the element that triggered the dialog.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button contrasts meet WCAG 2.1 AA standards. The icons for standard and destructive variants also provide a non-color-based distinction.
