Responsive Specifications

Desktop Layout

  • Standard layout with avatar, text content, and timestamp.
  • Text truncation with ellipsis is used for long notification messages to maintain layout integrity.

Tablet Adaptations

  • Layout remains consistent with desktop.
  • Padding and touch targets are maintained.

Mobile Adaptations

  • Layout remains consistent.
  • Font sizes may scale down based on the root typography settings.
  • Ensures the component is fully tappable and does not overflow the viewport.

Accessibility Features

Keyboard Navigation

Each notification item is a single focusable link, allowing users to tab through the list efficiently.

Screen Reader Support

The entire item is a link with an accessible name combining the notification text and state (e.g., 'Unread: Jane Doe sent you a connection request'). The unread indicator dot is hidden from screen readers to avoid noise.

Focus Management

Standard browser focus indicators are enhanced with the design system's focus ring on keyboard navigation.

Aria Attributes

  • aria-label for providing a comprehensive description of the notification link.
  • aria-hidden='true' for decorative elements like the unread dot.

Color Contrast

Text colors and background colors for both read and unread states meet WCAG 2.1 AA contrast requirements.
