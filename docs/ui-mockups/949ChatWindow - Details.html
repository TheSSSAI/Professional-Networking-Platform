Description

A self-contained organism for a one-on-one direct message conversation. It includes a header with user information, a scrollable message history, a message input form, and states for loading and disconnection.

Design System Alignment

This component is composed of several molecular and atomic components like UserBadge, MessageBubble, and form controls. It heavily utilizes the design system's tokens for color, typography, spacing, and shadows to create a cohesive and consistent user experience.

Responsive Specifications

Mobile Adaptations

  • Component takes up 100% of the viewport height and width, typical for a dedicated messaging screen on mobile.

Tablet Adaptations

  • Component becomes a windowed element with a max-height, border, and shadow, suitable for a two-pane layout (ConversationList + ChatWindow).

Desktop Enhancements

  • Same as tablet; provides a focused, windowed chat experience within a larger application layout.

Accessibility Features

Keyboard Support

The message input area is focusable and submittable with 'Enter'. Action buttons in the header are keyboard-focusable.

Screen Reader Support

The main message area has `role="log"` and new messages should be announced via an `aria-live` region (requires JS). The header `UserBadge` link provides context. Input has an associated label.

Focus Management

Focus is managed within the form elements in the footer. A clear `:focus-visible` state is applied to interactive elements.

Aria Attributes

  • role="log"
  • aria-label for the main chat region
  • aria-live for new messages
  • aria-label for icon buttons

Color Contrast

All text and background color combinations meet WCAG 2.1 AA contrast standards.
