Responsive Specifications

Desktop Layout

  • Modal is centered vertically and horizontally within the viewport.
  • A semi-transparent backdrop covers the entire screen.
  • The modal has a maximum width to ensure readability on large screens.

Tablet Adaptations

  • Layout is identical to desktop, with sufficient padding and touch-friendly button sizes.

Mobile Adaptations

  • Modal anchors to the bottom of the screen, full-width.
  • Padding is adjusted for smaller screens.
  • The backdrop still covers the full screen.

Accessibility Features

Keyboard Navigation

Full keyboard navigation is required. When open, focus must be trapped within the modal. The 'Escape' key must close the modal.

Screen Reader Support

The modal must use ARIA roles (`dialog`, `modal`). The title should be programmatically associated using `aria-labelledby`, and the description with `aria-describedby`.

Focus Management

When the modal opens, focus should move to the first focusable element inside it (e.g., the cancel button). When it closes, focus must return to the element that triggered it.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button contrasts must meet WCAG 2.1 AA standards. The destructive action button uses a high-contrast red.
