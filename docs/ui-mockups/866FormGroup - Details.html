A foundational form component that encapsulates a label, an input field, and an error message, ensuring proper structure, styling, and accessibility for form elements.

Responsive Specifications

Mobile Adaptations

  • The vertical stacking is inherently mobile-friendly.
  • Component will naturally take up the width of its container.

Tablet Adaptations

  • No specific changes needed; maintains vertical layout.

Desktop Enhancements

  • No specific changes needed; layout is determined by the parent form.

Accessibility Features

Keyboard Support

Standard keyboard interaction for form inputs (Tab, Shift+Tab).

Screen Reader Support

The `

Focus Management

The input field has a clear and compliant focus indicator.

Aria Attributes

  • `aria-describedby` links the input field to its error message, ensuring screen readers announce the error context.
  • `aria-invalid='true'` is applied to the input field in the error state.

Color Contrast

Label text, input text, and error message text meet WCAG 2.1 AA contrast requirements against their backgrounds.
