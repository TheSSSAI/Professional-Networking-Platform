Description

A comprehensive data table for displaying collections of data, such as user lists, moderation queues, or audit logs. It includes features like sorting, pagination, contextual actions, and various states for loading, empty, and error scenarios.

Design System Alignment

This organism component is composed of several atomic and molecular components, including Card, Button, Icon, Pagination, SkeletonLoader, and ActionMenu. It heavily relies on the design system's tokens for typography, color, spacing, and shadows to ensure a cohesive and consistent user experience.

Responsive Specifications

Mobile Adaptations

  • The primary adaptation for screens smaller than 768px is enabling horizontal scrolling within the `.table-wrapper`.
  • Cell padding is slightly reduced to conserve space.
  • Cell content is set to `white-space: nowrap` to prevent awkward text wrapping and ensure a consistent horizontal scroll experience.

Tablet Adaptations

  • The full desktop view is typically maintained, but horizontal scrolling will activate if the table content exceeds the viewport width.

Desktop Enhancements

  • The table is displayed in its full, non-scrolling layout.
  • Hover states on rows and sortable headers provide clear interaction feedback.
