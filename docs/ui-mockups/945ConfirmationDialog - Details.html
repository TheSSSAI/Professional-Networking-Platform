Description

A modal dialog that prompts the user to confirm a specific action. It is used to prevent accidental actions, especially for destructive operations like deletion. It has standard and destructive variants.

Design System Alignment

This component is an organism built upon the atomic 'Modal' and 'Button' components. It utilizes the design system's tokens for color (semantic, primary, neutral), spacing, typography, shadows, and border-radius to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The modal dialog is constrained to the viewport width with small margins.
  • The footer buttons stack vertically for easier touch interaction on narrow screens.

Tablet Adaptations

  • The dialog maintains its max-width, appearing as a centered modal.
  • Footer buttons are displayed horizontally.

Desktop Enhancements

  • The dialog is a compact, centered modal, allowing the background context to be visible.
  • Hover and focus states on buttons are fully functional.

Accessibility Features

Keyboard Support

The dialog itself and all buttons within it are keyboard accessible. In a real implementation, 'Escape' would close the dialog, and focus would be trapped within it while open.

Screen Reader Support

The dialog uses `role="alertdialog"`, which is a more assertive version of a standard dialog, appropriate for confirmations. It is labeled by its title via `aria-labelledby` and described by its body content via `aria-describedby`.

Focus Management

A clear, visible focus ring is applied to all interactive elements. In a real implementation, JavaScript would be required to manage focus trapping and return focus to the trigger element when the dialog is closed.

Aria Attributes

  • role="alertdialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet or exceed WCAG 2.1 AA contrast requirements.
