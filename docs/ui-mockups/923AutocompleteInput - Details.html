Description

An input field that provides users with a list of suggested options as they type. It's used for selecting values from a predefined or dynamically fetched list, such as skills, locations, or companies.

Design System Alignment

This component utilizes the design system's `TextInput` as its base, inheriting styles for default, focus, and error states. It uses the primary color for focus states, neutral colors for borders and backgrounds, and semantic error colors. Spacing tokens are used for padding and margins, while shadow and border-radius tokens are used for the dropdown menu.

Responsive Specifications

Mobile Adaptations

  • The suggestion dropdown menu becomes full-width on mobile viewports (< 768px) to maximize space and improve usability on smaller screens.

Tablet Adaptations

  • The component retains its desktop appearance and behavior, as there is sufficient screen real estate.

Desktop Enhancements

  • The dropdown menu is sized to the width of the input field, creating a clear visual connection.
  • Hover states on dropdown items provide clear feedback for mouse users.

Accessibility Features

Keyboard Support

The input is focusable with 'Tab'. Users should be able to navigate suggestions with 'Up'/'Down' arrows, select with 'Enter', and dismiss with 'Escape'. (Note: Full keyboard navigation requires JavaScript).

Screen Reader Support

The wrapper should have `role="combobox"`. The input should have `aria-autocomplete="list"` and `aria-expanded` to indicate the dropdown's state. The list should be `role="listbox"` and its items `role="option"`. `aria-activedescendant` on the input should point to the active suggestion's ID.

Focus Management

Focus remains on the input field while the dropdown is open. `:focus-within` is used in this static demo to manage the dropdown visibility based on focus.

Aria Attributes

  • role="combobox"
  • aria-autocomplete="list"
  • aria-expanded
  • aria-controls
  • aria-labelledby
  • aria-activedescendant
  • role="listbox"
  • role="option"

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast ratios.
