Responsive Specifications

Desktop Layout

  • The modal dialog is vertically and horizontally centered within the viewport.
  • A semi-transparent overlay covers the entire background.
  • The dialog has a maximum width (e.g., 450px) to maintain readability.

Tablet Adaptations

  • Maintains the desktop layout and sizing.
  • Ensures button touch targets are sufficiently large.

Mobile Adaptations

  • The dialog takes up most of the screen width, with small horizontal margins.
  • The dialog is positioned closer to the top of the viewport rather than being perfectly centered.
  • Typography and spacing are slightly reduced to fit smaller screens.

Accessibility Features

Keyboard Navigation

When open, focus is trapped within the modal. The 'Tab' key cycles through the interactive elements (e.g., Cancel button, Confirm button). The 'Escape' key closes the modal.

Screen Reader Support

The component uses `role='dialog'`, `aria-modal='true'`, `aria-labelledby` (pointing to the title), and `aria-describedby` (pointing to the description) to provide full context to screen reader users.

Focus Management

Upon opening, initial focus is programmatically set on the least destructive action (the 'Cancel' button) to prevent accidental confirmation.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet or exceed WCAG 2.1 AA contrast ratio requirements.
