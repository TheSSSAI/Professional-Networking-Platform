Description

A component to display a user's avatar and name, with an optional variant to include their professional headline. It is designed to be a single, clickable target that navigates to a user's profile.

Design System Alignment

This component uses design system tokens for colors (neutrals), typography (primary font family, body/caption sizes, weights), spacing, and border-radius. It is composed of the 'Avatar' atomic component.

Responsive Specifications

Mobile Adaptations

  • The component relies on its parent container's width. The `max-width` and text truncation properties ensure it adapts gracefully to smaller spaces without breaking layout.

Tablet Adaptations

  • Same as mobile; behavior is consistent across viewports.

Desktop Enhancements

  • A subtle hover effect is applied to indicate interactivity.

Accessibility Features

Keyboard Support

The entire component is a single anchor tag (``), making it a single tab stop. It is activatable with 'Enter'.

Screen Reader Support

The component is a link, so screen readers announce it as such. The text content (name and headline) provides the accessible name for the link.

Focus Management

A clear, high-contrast focus ring is applied on keyboard focus, using the design system's focus token.

Aria Attributes

  • The `alt` attribute on the avatar image provides context for screen reader users.

Color Contrast

Text colors (`--color-neutrals-gray-900` and `--color-neutrals-gray-600`) have sufficient contrast against the white/light gray background, meeting WCAG AA standards.
