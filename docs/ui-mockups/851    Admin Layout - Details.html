Responsive Specifications

Desktop Layout

  • Fixed sidebar on the left, fixed header at the top.
  • Main content area takes up the remaining space and is vertically scrollable.

Tablet Adaptations

  • Maintains the desktop layout, ensuring navigation links have adequate touch target size.

Mobile Adaptations

  • Sidebar is hidden off-screen by default.
  • A hamburger menu icon in the header toggles the sidebar's visibility.
  • When the sidebar is open, it slides in from the left and an overlay covers the main content area.

Accessibility Features

Keyboard Navigation

Logical tab order from header controls, to sidebar navigation, to the main content area. Sidebar is keyboard-operable on mobile.

Focus Management

When the mobile navigation is opened, focus is trapped within the sidebar. When closed, focus is returned to the hamburger menu button.

Aria Attributes

  • aria-label for icon-only buttons
  • aria-current='page' for the active navigation link
  • aria-expanded and aria-controls for the mobile navigation toggle

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
