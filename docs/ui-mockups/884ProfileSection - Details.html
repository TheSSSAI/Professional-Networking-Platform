Description

A container (Card) that displays a specific section of a user's profile, such as 'Work Experience', 'Education', or 'Skills'. It supports different content types and states (view, edit, empty) and includes controls for management when in an editable state.

Design System Alignment

This component is built upon the atomic 'Card' component for its base structure. It utilizes 'Button' and 'Icon' atoms for actions. All styling for typography, spacing, colors, and borders is derived from the design system's token library.

Responsive Specifications

Mobile Adaptations

  • The component is fluid and adapts to its container width.
  • List items remain vertically stacked, which is ideal for mobile viewports.

Tablet Adaptations

  • No specific adaptations are required; the component scales with the container.

Desktop Enhancements

  • Wider layouts allow for more content to be displayed without truncation.
  • Hover states on action buttons provide clear interaction feedback.

Accessibility Features

Keyboard Support

All action buttons (Add, Edit, Remove) are focusable and activatable using the keyboard (Tab, Enter/Space).

Screen Reader Support

The component is wrapped in a `

` with `aria-labelledby` pointing to the section title, providing clear context. Action buttons use `aria-label` to provide specific context (e.g., 'Add new work experience').

Focus Management

Focus indicators are consistent with the design system, ensuring keyboard users can easily track their position.

Aria Attributes

  • aria-labelledby
  • aria-label

Color Contrast

All text and background color combinations meet WCAG 2.1 AA contrast requirements.
