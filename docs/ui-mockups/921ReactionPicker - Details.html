Description

An interactive control allowing users to select from a set of reactions (e.g., Like, Celebrate) for a piece of content. It features a primary action button and a popover with extended options, triggered on hover for desktop and long-press for touch devices.

Design System Alignment

Uses design system tokens for colors (primary, neutral), spacing, shadows, border-radius, and typography. The trigger button aligns with the 'text' button variant, and the popover uses the standard card shadow.

Responsive Specifications

Mobile Adaptations

  • The popover is intended to be triggered via long-press on touch devices, a behavior simulated here with a static CSS class.
  • The component's size is compact and works well on mobile without layout changes.

Tablet Adaptations

  • Same as mobile, with touch interaction being the primary model.

Desktop Enhancements

  • Popover is triggered on hover for quick access.
  • Keyboard accessibility is supported via :focus-within to open the popover.
  • Tooltips appear on hover over individual reactions to clarify their meaning.

Accessibility Features

Keyboard Support

The main trigger button is focusable (Tab). Once focused, the popover appears, and users can Tab through the individual reaction options. Enter/Space activates a reaction.

Screen Reader Support

The main trigger button should have an `aria-label` like 'React to post'. Each reaction option button has a descriptive `aria-label`.

Focus Management

Using `:focus-within` on the parent container keeps the popover open while focus is on any of its children, which is a robust pattern for keyboard navigation.

Aria Attributes

  • aria-label for all icon-only buttons.
  • aria-haspopup="true" on the trigger button to indicate it opens a menu.
  • aria-expanded="false/true" would be added via JavaScript to track the state programmatically.

Color Contrast

Text and icon colors meet WCAG AA contrast ratios against their backgrounds.
