Description

A navigation component for traversing paginated content, such as lists of users, posts, or moderation queue items. It adapts to a compact view on mobile devices.

Design System Alignment

This component utilizes the design system's color palette for states (primary, neutral), spacing tokens for padding and gaps, and border-radius tokens for consistent shape. It also inherits font styles.

Responsive Specifications

Mobile Adaptations

  • Component collapses to a 'simple' view showing only 'Previous' and 'Next' buttons for a compact interface.
  • Ensures touch targets remain at least 44x44px.

Tablet Adaptations

  • Expands to the 'standard' view, showing page numbers, the current page, and ellipses for larger ranges.

Desktop Enhancements

  • Full 'standard' view with clear hover and focus states for mouse and keyboard navigation.

Accessibility Features

Keyboard Support

All interactive elements (buttons, links) are focusable with Tab and activatable with Enter/Space.

Screen Reader Support

The component is wrapped in a

landmark with an `aria-label`. The current page is marked with `aria-current="page"`. Previous/Next buttons have screen-reader-only text for context.

Focus Management

A clear, high-contrast focus ring is applied to all interactive elements on `:focus-visible`.

Aria Attributes

  • aria-label="pagination"
  • aria-current="page"
  • aria-label="Go to previous page"
  • aria-label="Go to next page"
  • aria-label="Go to page X"

Color Contrast

Active and default states meet WCAG 2.1 AA contrast requirements.
