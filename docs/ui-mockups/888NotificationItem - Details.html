Description

A single item within a notification list. It displays the actor, action, and timestamp, functioning as a link to the relevant content. It has 'read' and 'unread' states and variants for different notification types.

Design System Alignment

This component uses design system tokens for colors, typography, spacing, and border-radius. It is composed of atomic components like Avatar and Typography wrappers, and molecular components like Buttons for actions.

Responsive Specifications

Mobile Adaptations

  • The component is fluid and adapts to its container's width.
  • Text content will wrap as needed, but the main line of text is not designed for multi-line truncation.

Tablet Adaptations

  • No specific adaptations required; layout is consistent.

Desktop Enhancements

  • Hover states provide clear visual feedback.

Accessibility Features

Keyboard Support

The entire component is a single tab stop as it's wrapped in an `` tag. Action buttons within variants are also focusable.

Screen Reader Support

The content of the notification (user name, action, timestamp) serves as the accessible name for the link. The `unread` state is announced via visually hidden text.

Focus Management

A clear, high-contrast focus ring is applied to the entire component and any internal interactive elements.

Aria Attributes

  • No specific ARIA attributes are needed for the base component as it uses semantic HTML (`` and `
  • `). Internal buttons have `aria-label` where necessary.

Color Contrast

Text and background colors for both read and unread states meet WCAG 2.1 AA contrast requirements.
