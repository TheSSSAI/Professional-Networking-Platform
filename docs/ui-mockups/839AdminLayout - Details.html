Description

The main shell for the entire admin dashboard application. It provides a consistent structure with a persistent sidebar for navigation, a header for user information and actions, and a main content area for specific pages.

Design System Alignment

This organism orchestrates molecular components like AdminHeader and SidebarNav. It uses design system tokens for layout structure, such as sidebar width, header height, spacing, and background colors.

Responsive Specifications

Mobile Adaptations

  • Sidebar is hidden off-screen by default.
  • A hamburger menu icon in the header toggles the sidebar's visibility.
  • When the sidebar is open, it slides in from the left and an overlay covers the main content area.

Tablet Adaptations

  • The mobile layout (hidden sidebar) is typically used for tablets in portrait mode.

Desktop Enhancements

  • Switches to a two-column grid layout at 1024px.
  • The sidebar becomes permanently visible and fixed on the left.
  • The header and main content area occupy the remaining space.

Accessibility Features

Keyboard Support

The mobile navigation toggle is keyboard-focusable. All navigation items within the sidebar and header are keyboard accessible. The overlay can be closed with the Escape key (requires JS). A 'Skip to main content' link is included for keyboard users.

Focus Management

Visible focus rings are applied to all interactive elements. For the mobile navigation, JavaScript would be required to trap focus within the open sidebar and return it to the toggle button when closed.
