Description

A self-contained form organism for user authentication. It includes fields for email and password, a submit button, a link for password recovery, and states for form submission and error feedback.

Design System Alignment

This organism is composed of molecular (FormGroup) and atomic (TextInput, Button, Alert) components. It heavily relies on the design system's tokens for color, spacing, typography, and borders to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The form takes up the full width of its container to be easily usable on small screens.
  • Maintains vertical rhythm with adequate spacing for touch targets.

Tablet Adaptations

  • No specific adaptations needed, continues to scale with the container width.

Desktop Enhancements

  • The form is presented within a card with a max-width and box-shadow, centered on the page for a focused experience.

Accessibility Features

Keyboard Support

The form is fully navigable and operable using a keyboard. Users can tab between fields and the submit button. Pressing Enter in the password field submits the form.

Screen Reader Support

Each input is correctly associated with a `

Focus Management

All interactive elements have a clear and visible focus state that meets WCAG 2.1 AA contrast requirements.

Aria Attributes

  • role='alert' for the error message container to ensure it's announced by screen readers.

Color Contrast

All text and UI elements meet the minimum 4.5:1 contrast ratio for normal text.
