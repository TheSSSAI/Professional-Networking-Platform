Description

A self-contained form for handling the Multi-Factor Authentication (MFA) step in a login flow. It prompts the user for a 6-digit code from their authenticator app and includes states for submission and errors.

Design System Alignment

This organism is composed of molecular (FormGroup) and atomic (TextInput, Button, Alert) components. It strictly adheres to the design system's tokens for colors, typography, spacing, and border radii to ensure visual consistency.

Responsive Specifications

Mobile Adaptations

  • The form is fluid and takes up available width, with appropriate padding.
  • Maintains a single-column, centered layout which is inherently mobile-friendly.

Tablet Adaptations

  • The form maintains its centered layout with a max-width to ensure readability and prevent it from becoming too wide.

Desktop Enhancements

  • The card-based layout with shadows is more pronounced on desktop, providing clear visual hierarchy.

Accessibility Features

Keyboard Support

All interactive elements (input field, submit button) are fully navigable and operable using the Tab and Enter/Space keys.

Screen Reader Support

Focus Management

A clear, visible focus ring (`--focus-ring`) is applied to both the input field and the submit button, meeting WCAG requirements.

Aria Attributes

  • `aria-invalid` is used on the input field in the error state.
  • `aria-describedby` links the input field to its error message.
  • `role='alert'` is used on the form-level error message container.

Color Contrast

All text and UI elements meet or exceed the WCAG 2.1 AA contrast ratio of 4.5:1 for normal text.
