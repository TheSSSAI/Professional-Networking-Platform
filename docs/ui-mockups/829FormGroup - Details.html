Description

A molecular component that groups a label, a form input, and an optional error message. It manages the layout, spacing, and accessibility bindings between these elements, and provides a clear error state.

Design System Alignment

Utilizes design system tokens for spacing, typography, and semantic colors (error state). It combines atomic components like TextInput, Label, and ErrorMessage into a reusable, functional unit.

Responsive Specifications

Mobile Adaptations

  • The component naturally stacks vertically, which is ideal for mobile viewports.
  • Ensures full-width usage within its parent container on small screens.

Tablet Adaptations

  • Maintains the vertical stacking layout, adjusting to container width.

Desktop Enhancements

  • No specific desktop enhancements are required; the component is designed to be flexible within larger form layouts.

Accessibility Features

Keyboard Support

The child input element is fully focusable and operable via keyboard.

Screen Reader Support

The `

Focus Management

The child input element receives a clear, high-contrast focus ring.

Aria Attributes

  • aria-invalid="true" is applied to the input in the error state.
  • aria-describedby is used to link the input to its corresponding error message.

Color Contrast

Label text, input text, and error text all meet WCAG AA contrast ratios against their backgrounds.
