Description

A UserBadge combines a user's Avatar and their identity (Name, and optionally Headline) into a single, reusable component. It typically acts as a link to the user's profile.

Design System Alignment

This component uses design system tokens for avatars, spacing, typography (font sizes, weights), and colors to ensure consistency with the overall visual foundation.

Responsive Specifications

Mobile Adaptations

  • The component is fluid by nature and will adapt to its container.
  • Text truncation prevents long names or headlines from breaking layouts on small screens.

Tablet Adaptations

  • No specific adaptations needed.

Desktop Enhancements

  • Hover effect on the user's name to indicate interactivity.

Accessibility Features

Keyboard Support

The entire component is a single link, making it a single tab stop. It is activatable with 'Enter'.

Screen Reader Support

As a semantic link (``), screen readers will announce it as 'Link, [User Name], [User Headline]'. The `` for the avatar has an `alt` attribute.

Focus Management

A clear, high-contrast focus ring is applied when the component is focused using a keyboard.

Aria Attributes

  • The avatar image uses `alt` text for screen readers.

Color Contrast

Text colors (`--color-neutral-900` and `--color-neutral-600`) meet WCAG AA contrast ratios against the light background.

Implementation Notes

Embedded Css Architecture

Component-specific CSS is defined using BEM methodology. It relies on a base set of design system tokens (CSS custom properties) for styling.

Component Organization

The HTML structure uses an `` tag as the root element to ensure the entire badge is a single interactive target. Inside, it composes the `Avatar` and text elements.

Responsive Design

Responsiveness is achieved through text truncation and a flexible `flex` layout, allowing the component to fit within various parent containers without breaking.

Accessibility Compliance

Semantic HTML (``, ``) is used to provide a good baseline. Focus styles and alt text are included to meet WCAG 2.1 AA requirements.

Browser Support

Built with modern CSS (flexbox, custom properties) compatible with all evergreen browsers.

Usage Guidelines

When To Use

Use whenever attributing content or an action to a user, such as in post headers, comment threads, search results, and notification items.

Content Guidelines

The name and headline should be sourced directly from the user's profile data. Text should be truncated if it overflows the available space.

Placement Rules

Typically placed at the top of a content card (like a post) or as the primary element in a list item (like a notification).

Customization Options

  • Use `.user-badge--small` for a more compact version that only shows the name.
  • The default class (`.user-badge`) provides the larger version with the headline.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, Edge.

Device Testing

Test on mobile and desktop viewports to ensure text truncation works correctly and doesn't break layouts.

Accessibility Testing

  • Keyboard navigation: ensure the badge is a single, focusable tab stop.
  • Screen reader testing: verify that the name and headline are read out as part of the link text.
  • Verify the focus indicator is visible and has sufficient contrast.
