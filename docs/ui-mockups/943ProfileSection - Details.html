Description

A self-contained card for displaying a section of a user's profile, such as Work Experience, Education, or Skills. It includes states for viewing, editing, and when the section is empty.

Design System Alignment

This organism component is built upon the atomic 'Card', 'Button', and 'Icon' components. It uses the design system's tokens for spacing, typography, colors, border-radius, and shadows to create a consistent and modular profile layout.

Responsive Specifications

Mobile Adaptations

  • The vertical stacking of items is inherently mobile-friendly.
  • Padding within the card is reduced slightly on smaller screens to maximize content space.

Tablet Adaptations

  • Maintains the default vertical stacking layout.

Desktop Enhancements

  • The layout is designed for a single column within a multi-column page layout, so its internal structure remains consistent.

Accessibility Features

Keyboard Support

All interactive elements (Add, Edit, Done buttons, and individual item edit/delete buttons) are focusable with Tab and activatable with Enter/Space.

Screen Reader Support

The component uses a semantic `

` tag with a heading (`

`) for proper document structure. Icon-only buttons have descriptive `aria-label` attributes.

Focus Management

A clear, high-contrast `:focus-visible` ring is applied to all interactive elements.

Aria Attributes

  • aria-label for icon buttons (e.g., 'Add new work experience', 'Edit experience at Innovate Corp').

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
