Description

A complex form component for creating user posts. It supports text, image uploads, and link previews. It can be displayed as an inline card or as a modal dialog and includes states for default, submitting, and error.

Design System Alignment

This organism is composed of multiple atomic and molecular components, including Card, Avatar, Button, FileUpload, and Alert. It heavily relies on the design system's tokens for color, typography, spacing, and shadows to maintain a cohesive look and feel.

Responsive Specifications

Mobile Adaptations

  • The modal variant becomes a full-screen component on viewports smaller than 768px to maximize editing space.
  • The inline variant adapts to the width of its container, typically a single column in a mobile feed.

Tablet Adaptations

  • The modal variant remains a centered dialog.
  • Layout is consistent with desktop.

Desktop Enhancements

  • The modal variant is a centered dialog with an overlay, providing a focused writing experience.
  • The inline variant is typically used as a primary call-to-action at the top of a feed.

Accessibility Features

Keyboard Support

All interactive elements (buttons, icons, textarea) are focusable and operable via keyboard. The modal traps focus when open.

Screen Reader Support

The textarea has a programmatically associated label (e.g., via `aria-label` or a visible `

Focus Management

When the modal is opened, focus is automatically moved to the textarea. When closed, focus returns to the trigger element.

Aria Attributes

  • aria-label
  • aria-describedby
  • aria-modal
  • role='dialog'

Color Contrast

All text and UI controls meet WCAG 2.1 AA contrast requirements.
