Description

A navigation component for paginated content, allowing users to move between pages. Includes standard and simple variants, with responsive adaptations for mobile viewports.

Design System Alignment

Utilizes design system tokens for colors (primary, neutrals), spacing, typography, and border-radius. Inherits button styles for interactive elements, ensuring consistency with atomic components.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 768px, the 'standard' pagination component hides the page number links, showing only the 'Previous' and 'Next' buttons to save space.
  • This effectively makes the standard variant behave like the simple variant on mobile.
  • Touch targets are maintained at a minimum of 40x40px.

Tablet Adaptations

  • The full 'standard' pagination component is displayed.

Desktop Enhancements

  • The full 'standard' pagination component with all page numbers and hover effects is displayed.

Accessibility Features

Keyboard Support

All links are focusable via Tab key and activatable via Enter key.

Screen Reader Support

The component is wrapped in a

element with an `aria-label='pagination'`. The current page is marked with `aria-current='page'`. Disabled links have `aria-disabled='true'`.

Focus Management

A clear, high-contrast focus-visible ring is applied to all interactive elements.

Aria Attributes

  • aria-label
  • aria-current
  • aria-disabled

Color Contrast

Active and default states meet WCAG 2.1 AA contrast requirements.
