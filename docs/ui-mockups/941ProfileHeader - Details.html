Description

An organism component that serves as the main header for a user's profile page. It displays the banner image, profile picture, user's name and headline, and primary action buttons. It has distinct variants for viewing one's own profile versus viewing another user's profile.

Design System Alignment

This component is a composition of several atomic and molecular components, including Avatar, Button, FileUpload, and ActionMenu. It heavily utilizes the design system's tokens for color, typography, spacing, and shadows to create a cohesive and branded experience.

Responsive Specifications

Mobile Adaptations

  • The main content area (avatar, info, actions) stacks vertically and is center-aligned.
  • Action buttons take up full width for easier tapping.
  • The avatar size is reduced.
  • The main content block's negative margin is reduced to suit the smaller avatar.

Tablet Adaptations

  • Switches to a row-based layout for the avatar, info, and actions.
  • The user info and actions align to the left and right respectively.

Desktop Enhancements

  • Utilizes the full row-based layout with ample spacing.
  • Hover effects on buttons and interactive elements are prominent.

Accessibility Features

Keyboard Support

All interactive elements (buttons, menus) are keyboard focusable and operable using Tab, Enter, and Space keys.

Screen Reader Support

Semantic HTML (`

Focus Management

Visible focus rings are applied to all interactive elements using `:focus-visible`.

Aria Attributes

  • `aria-label` for icon buttons like 'Edit banner' or 'More actions'.
  • `alt` attributes for banner and profile images.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
