Description

A component for displaying a single message in a chat conversation. It has distinct visual styles for 'sent' and 'received' messages and includes status indicators for sending, sent, delivered, and read states.

Design System Alignment

This component uses design system tokens for colors (primary, neutrals), typography (body and caption sizes), spacing (for padding and margins), and border-radius to create the bubble shape.

Responsive Specifications

Mobile Adaptations

  • The component is intrinsically responsive, adapting its width based on the `max-width` percentage and the length of its content. No media queries are required.

Tablet Adaptations

  • Same as mobile; behavior is consistent across viewports.

Desktop Enhancements

  • The tooltip for status indicators is primarily a desktop enhancement, activated on hover.

Accessibility Features

Keyboard Support

Not applicable as the component is non-interactive by default. The tooltip is a progressive enhancement and not keyboard-focusable to avoid cluttering the tab order in a long chat.

Screen Reader Support

Message content is read as standard text. The status indicator icon has a visually hidden `` to provide an accessible name (e.g., 'Message delivered').

Focus Management

Not applicable.

Aria Attributes

  • No specific ARIA attributes are required for the base component. The visually hidden text provides necessary context.

Color Contrast

Text color and background color combinations for both 'sent' and 'received' variants meet WCAG 2.1 AA contrast requirements.
