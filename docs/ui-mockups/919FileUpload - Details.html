Description

A component for handling file selection and upload, providing visual feedback for idle, uploading, success, and error states. It supports single-file (profile, banner) and multi-file (post images) variants.

Design System Alignment

Utilizes the design system's color palette for states (primary for focus, error for validation), spacing tokens for padding and margins, and border-radius tokens for consistency with other components like Cards and Inputs.

Implementation Notes

Embedded Css Architecture

All styles are embedded, using CSS custom properties for design tokens. BEM-style classes (`file-upload__prompt`) are used for modularity. State is managed via a `data-state` attribute on the root component.

Component Organization

The showcase is organized by component variant, with each state demonstrated statically. This allows for easy visual regression testing and review.

Responsive Design

The component is intrinsically responsive, using flexbox, grid, and aspect-ratio to adapt to its container. No explicit breakpoints are needed for the component itself.

Accessibility Compliance

The core structure uses a semantic `label` and a visually-hidden `input` for built-in accessibility. JavaScript is required to announce dynamic state changes using ARIA live regions.

Browser Support

Built for modern evergreen browsers (Chrome, Firefox, Safari, Edge). The use of `:focus-within` and `aspect-ratio` is well-supported.

Usage Guidelines

When To Use

Use for any user-initiated file upload, including profile pictures, banners, or images for posts.

Content Guidelines

Prompt text should be concise and action-oriented. Metadata text should clearly state file type and size limitations.

Placement Rules

Place inline with forms or as a primary interaction point on a profile or composer.

Customization Options

  • Use `.file-upload--profile` for a circular, avatar-style uploader.
  • Use `.file-upload--banner` for a wide, rectangular banner uploader.
  • Use the `.file-upload--multi-container` structure for allowing multiple file uploads.

Testing Recommendations

Browser Testing

Test in Chrome, Firefox, Safari, Edge.

Device Testing

Test on mobile (touch interaction) and desktop (mouse drag-and-drop and click).

Accessibility Testing

  • Keyboard navigation to and activation of the upload control.
  • Screen reader verification of labels, states, and error messages (requires JS for ARIA updates).
  • Color contrast validation for all states.

Performance Testing

Not applicable at the static component level, but the consuming application should monitor performance during actual file uploads.
