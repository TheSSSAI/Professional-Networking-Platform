Responsive Specifications

Desktop Layout

  • Contained within a max-width for readability in a multi-column feed layout.
  • Maintains standard spacing and font sizes as per the design system.

Tablet Adaptations

  • Layout remains similar to desktop but may occupy a larger percentage of the screen width.

Mobile Adaptations

  • Component spans the full width of the viewport with reduced horizontal margins.
  • Spacing and font sizes are reduced slightly for better density on smaller screens.
  • Touch targets for all interactive elements are a minimum of 44x44px.

Accessibility Features

Keyboard Navigation

All interactive elements (author link, action menu, reaction buttons, content links) are focusable and operable via keyboard in a logical order.

Screen Reader Support

The component is wrapped in an

tag. The author, content, and actions are structured semantically. Images have alt text and icon-only buttons have aria-labels.

Focus Management

A clear, visible focus state is applied to all interactive elements.

Aria Attributes

  • aria-label for icon buttons
  • aria-pressed for toggle buttons (e.g., Like)
  • role attributes for custom components like menus

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
