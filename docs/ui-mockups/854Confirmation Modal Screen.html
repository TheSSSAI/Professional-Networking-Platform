Screen Variations

This section documents the different states and variants of the component. The modals above are interactive examples.

Variant Name: "Closed State"

The modal is not visible on the screen. This is the default state before a user action triggers it. No HTML is rendered, or it is hidden with `display: none`.

Variant Name: "Standard Action"

Used for confirmations that are not destructive, such as saving changes. The primary action button uses the standard primary style.

Variant Name: "Destructive Action"

Used for irreversible actions like deletion. The confirmation button uses the destructive (danger) style to draw attention to the severity of the action.
