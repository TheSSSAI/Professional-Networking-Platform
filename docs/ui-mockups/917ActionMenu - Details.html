Description

A context-sensitive menu component that provides a list of actions for a specific item, such as a post or a comment. It is triggered by an icon button and displays a dropdown list of options.

Design System Alignment

Uses the design system's tokens for colors (neutrals, primary, semantic-error), spacing, shadows, and border-radius to ensure visual consistency with other components. It relies on the atomic Button and Icon components for its construction.

Responsive Specifications

Mobile Adaptations

  • No specific adaptations needed. The component's use of absolute positioning for the dropdown works inherently across viewport sizes.
  • The trigger button maintains a minimum 44x44px touch target size.

Tablet Adaptations

  • No specific adaptations needed.

Desktop Enhancements

  • Hover states are fully functional on mouse-based devices.

Accessibility Features

Keyboard Support

The trigger button is focusable with Tab. The menu opens on Enter/Space. Arrow keys can navigate menu items. Esc can close the menu (browser/JS dependent). The :focus-within implementation ensures the menu stays open during keyboard navigation.

Screen Reader Support

The trigger button has an `aria-label` providing context. `aria-haspopup='true'` indicates it controls a menu. The dropdown can be semantically marked with `role='menu'` and items with `role='menuitem'` for screen readers.

Focus Management

Focus is naturally managed by the browser. When the menu opens, focus moves to the first item. When focus leaves the component, the menu closes.

Aria Attributes

  • aria-label
  • aria-haspopup
  • aria-expanded

Color Contrast

Text and icons meet WCAG 2.1 AA contrast requirements against their backgrounds.
