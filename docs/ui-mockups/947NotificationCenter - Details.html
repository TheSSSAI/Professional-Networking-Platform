Description

A dropdown/popover panel that provides a user with a historical list of their recent notifications. It includes loading and empty states, and visually distinguishes between read and unread items.

Design System Alignment

This component is composed of several atomic and molecular components, including Avatar, Badge, LoadingSpinner, and the custom NotificationItem. It uses the design system's tokens for colors, spacing, typography, shadows, and border-radius to create a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • On viewports narrower than 480px, the panel expands to nearly the full width of the screen to ensure readability.

Tablet Adaptations

  • The component maintains its desktop width and positioning.

Desktop Enhancements

  • The component is a fixed-width popover, optimized for quick scanning.

Accessibility Features

Keyboard Support

The bell icon toggle is keyboard focusable and can be activated with Enter/Space. Once open, users can Tab through the interactive elements (links, notifications). The panel can be closed with the Escape key (requires JS).

Screen Reader Support

The toggle button has `aria-haspopup` and `aria-expanded` attributes. The panel has `role='region'` and an appropriate `aria-label`. Unread items are visually distinct, and this state should be conveyed via `aria-label` on the item.

Focus Management

Focus indicators are provided for all interactive elements. A pure CSS `:focus-within` is used to keep the panel open during keyboard navigation for this static mockup.

Aria Attributes

  • aria-haspopup="true"
  • aria-expanded="false/true"
  • aria-label="Notifications"
  • role="region"

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
