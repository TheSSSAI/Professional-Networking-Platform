Responsive Specifications

Desktop Layout

  • Centered horizontally and vertically on the screen.
  • Max-width is set to prevent the dialog from becoming too wide on large screens.
  • A semi-transparent overlay covers the entire background.

Tablet Adaptations

  • Maintains the desktop's centered layout.
  • Ensures buttons are large enough for easy touch interaction.

Mobile Adaptations

  • Dialog width increases to use more of the screen space.
  • Margins around the dialog are reduced.
  • Font sizes and padding may be slightly reduced to fit content comfortably on smaller screens.

Accessibility Features

Keyboard Navigation

When open, focus is trapped within the dialog. Users can tab between the title, content, and action buttons. The 'Escape' key closes the dialog.

Screen Reader Support

The dialog uses the semantic `

` element or has `role="alertdialog"`. The title of the dialog is programmatically associated using `aria-labelledby`, and the descriptive text is associated with `aria-describedby`.

Focus Management

Upon opening, focus is automatically moved to the first focusable element (typically the 'Cancel' or less destructive button). Upon closing, focus is returned to the element that originally triggered the dialog.

Aria Attributes

  • aria-labelledby
  • aria-describedby
  • aria-modal="true"

Color Contrast

All text and button contrasts meet WCAG 2.1 AA requirements. The destructive action button uses a high-contrast red to signify its purpose.
