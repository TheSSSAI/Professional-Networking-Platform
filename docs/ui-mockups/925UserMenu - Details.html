Description

A composite UI control that combines a user's avatar with a dropdown menu for account-related navigation, such as viewing a profile, accessing settings, or logging out. It serves as a primary entry point for user account management within the main application header.

Design System Alignment

This component utilizes atomic components and design tokens from the system. It uses 'Avatar' for the user representation, 'Link' for navigation items, and a custom 'DropdownMenu' structure built with 'Card' principles (shadow, border-radius). It heavily relies on color, spacing, and typography tokens.

Accessibility Features

Keyboard Support

The toggle button is focusable with 'Tab' and activatable with 'Enter' or 'Space'. Menu items can be navigated with 'Tab' (or arrow keys with JavaScript). 'Escape' should close the menu (requires JS).

Screen Reader Support

The toggle button has `aria-haspopup='true'` and `aria-expanded` to announce its state. The dropdown has `role='menu'` and items have `role='menuitem'`.

Focus Management

A visible focus ring is applied to the toggle and menu items. For a production component, JavaScript would be needed to trap focus within the open menu and return focus to the toggle button when closed.

Aria Attributes

  • aria-label='Open user menu'
  • aria-haspopup='true'
  • aria-expanded='false'/'true'
  • role='menu'
  • role='menuitem'

Color Contrast

All text and background color combinations meet WCAG 2.1 AA standards.

Implementation Notes

Embedded Css Architecture

Styles are designed to be embedded, using BEM for scoping and CSS custom properties for tokenization. This mockup simulates the open state by adding an `.is-open` class.

Component Organization

Responsive Design

The component is intrinsically responsive and does not require specific media query adaptations. Its width is fixed, and it is positioned relative to its parent.

Accessibility Compliance

The static HTML/CSS provides the necessary semantic structure and ARIA roles. Full accessibility, including focus trapping and keyboard navigation (arrow keys, Esc), would require JavaScript.

Browser Support

Designed for modern evergreen browsers (Chrome, Firefox, Safari, Edge).

Usage Guidelines

When To Use

Use in the main application header or any persistent navigation bar to provide users with access to their account settings and logout functionality.

Content Guidelines

The user's full name and current professional headline should be displayed in the dropdown header. Menu item text should be concise and action-oriented (e.g., 'View Profile', 'Logout').

Placement Rules

Typically placed on the far right of the application header.
