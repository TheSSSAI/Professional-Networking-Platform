Description

A navigational component used in the messaging interface to display a list of a user's ongoing conversations. It supports loading, loaded, and empty states, and highlights the currently active conversation.

Design System Alignment

This component is composed of molecular components like UserBadge and SkeletonLoader. It heavily relies on the design system's tokens for colors (primary, neutral), spacing, typography, and border-radius to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The component is designed to be fluid and take up the full screen width in a single-pane messaging layout.
  • Touch targets for list items are large enough for easy interaction.

Tablet Adaptations

  • Behaves similarly to mobile in a single-pane layout or occupies a sidebar in a two-pane layout.

Desktop Enhancements

  • Typically used as a sidebar. Hover states provide clear feedback on interactive elements.

Accessibility Features

Keyboard Support

The list of conversations can be navigated using the Tab key, as each item is a link. 'Enter' activates the link.

Screen Reader Support

The component should be contained within a `

` with an `aria-label`. Each link should have a descriptive label, including the user's name and unread status, e.g., 'Conversation with Jane Doe, 1 unread message'.

Focus Management

A clear, high-contrast `:focus-visible` style is applied to each conversation link.

Aria Attributes

  • aria-label for the navigation region.
  • aria-current='page' for the active conversation link.
  • Visually hidden text for unread indicators.

Color Contrast

Text and background colors meet WCAG 2.1 AA contrast requirements for all states (default, active, hover).
