Description

A dropdown menu component triggered by the user's avatar, providing access to profile-related actions like viewing the profile, accessing settings, and logging out. It is typically placed in the main application header.

Design System Alignment

Utilizes design system tokens for colors (neutrals, primary), spacing, border-radius, and typography. It combines atomic components like Avatar (conceptual), Link, and follows dropdown interaction patterns.

Responsive Specifications

Mobile Adaptations

  • The component is inherently responsive and works well on mobile. The dropdown may have a max-width to avoid taking up the full screen width on very small devices.

Tablet Adaptations

  • No specific adaptations needed.

Desktop Enhancements

  • Hover states on menu items are more prominent on desktop.

Accessibility Features

Keyboard Support

The trigger is focusable with Tab. When open, menu items can be navigated with Tab. The menu can be opened/closed with Enter/Space on the trigger. It can be closed with the Escape key.

Screen Reader Support

Using a `

` and `` element pair provides semantic open/close state announcement. Links within the menu are announced as standard links.

Focus Management

Focus is naturally handled by the browser for the `

` element. A visible focus ring is applied to the trigger and menu items for keyboard users.

Aria Attributes

  • The `` element implicitly has `aria-expanded` and other relevant attributes, enhancing accessibility.
  • Menu items can be further enhanced with `role='menuitem'` within a `role='menu'` parent for stricter screen reader interpretation, though anchor tags are generally well-supported.

Color Contrast

All text and icons have a contrast ratio of at least 4.5:1 against their backgrounds, meeting WCAG AA standards.
