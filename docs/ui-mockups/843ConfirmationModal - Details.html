Description

A modal dialog that interrupts the user's workflow to request confirmation for a significant or destructive action. It includes a title, a descriptive message, and a set of actions (confirm, cancel).

Design System Alignment

This organism is composed of a Modal container, Typography for the title and message, and atomic Button components for actions. It heavily utilizes the design system's tokens for colors (primary, danger, neutral), spacing, shadows, and border-radius to ensure a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • The modal panel is nearly full-width with small side margins.
  • Action buttons in the footer stack vertically for easier tapping.

Tablet Adaptations

  • The modal has a fixed max-width, appearing centered on the screen.
  • Action buttons in the footer are arranged horizontally.

Desktop Enhancements

  • Maintains the tablet layout with a fixed max-width.

Accessibility Features

Keyboard Support

Fully keyboard navigable. 'Tab' cycles through focusable elements (close button, action buttons). 'Escape' key closes the modal (requires JS).

Screen Reader Support

The modal container has `role="dialog"` and `aria-modal="true"`. The title is linked via `aria-labelledby`, and the description is linked via `aria-describedby`.

Focus Management

For a production implementation, JavaScript is required to trap focus within the modal when it is open and return focus to the trigger element when it closes.

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby
  • aria-label for close button

Color Contrast

All text and button contrasts meet WCAG 2.1 AA requirements.
