Description

A self-contained organism component representing a single user-generated post in a news feed. It displays author information, content (text, images, or link previews), and action controls for engagement (reactions, comments).

Design System Alignment

This organism is composed of the 'Card' atomic component and several molecular components: 'UserBadge', 'ActionMenu', and 'ReactionPicker'. It heavily relies on the design system's tokens for typography, spacing, colors, and shadows to ensure a cohesive look and feel.

Responsive Specifications

Mobile Adaptations

  • The component is fluid and adapts to its container, which is expected to be a single column on mobile.
  • Padding and font sizes are inherited from the design system's responsive tokens, ensuring readability on smaller screens.

Tablet Adaptations

  • The component maintains its structure, adapting to the wider container width on tablets.

Desktop Enhancements

  • Hover states on interactive elements like the ActionMenu, ReactionPicker, and action buttons provide richer feedback on desktop.
  • The layout remains a single-column card, as is standard for feed content.

Accessibility Features

Keyboard Support

All interactive child components (UserBadge link, ActionMenu, ReactionPicker, Comment/Share buttons) are keyboard focusable and operable.

Screen Reader Support

The component is wrapped in an `

` tag for semantic meaning. The header, content, and footer are clearly structured. `aria-label` attributes are used on all icon-only buttons within child components.

Focus Management

Visible focus rings are applied to all interactive elements, consistent with the design system.

Aria Attributes

  • aria-label on icon buttons
  • aria-haspopup on menu triggers

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
