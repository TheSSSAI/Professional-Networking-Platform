Description

The main application header, persistent across the top of the screen. It provides global navigation, search, and access to user-specific actions. It has distinct variants for authenticated and guest users and is fully responsive.

Design System Alignment

This organism is a composition of molecular components: SearchBar, UserMenu, and atomic components like Logo, NavigationLinks, and NotificationBell. It uses the design system's grid, spacing, and color tokens to define its layout and appearance across different breakpoints.

Responsive Specifications

Mobile Adaptations

  • Navigation links are collapsed into a hamburger menu.
  • The global search bar is collapsed into an icon-only button.
  • The mobile navigation panel slides in from the side.

Tablet Adaptations

  • Maintains the mobile layout for consistency on touch devices.

Desktop Enhancements

  • Displays the full search bar and a horizontal list of navigation links.
  • Hides the mobile-specific hamburger and search icon triggers.

Accessibility Features

Keyboard Support

All interactive elements (logo, links, buttons, user menu) are keyboard-focusable and activatable. The mobile navigation toggle can be operated with Enter/Space.

Screen Reader Support

The header is a semantic `

` element. Navigation is wrapped in a `
`. The mobile navigation toggle button uses `aria-controls` and `aria-expanded` to communicate its state to screen readers.

Focus Management

Visible focus rings are applied to all interactive elements. When the mobile navigation is open, focus should be trapped within the panel (requires JavaScript).

Aria Attributes

  • aria-label for icon-only buttons (search, notifications, hamburger menu).
  • aria-controls and aria-expanded for the mobile navigation toggle.
  • aria-current='page' on the active navigation link.

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
