Description

A composite form control for user input to perform searches. It combines a text input with an icon and a submit button. It includes variants for loading states and an optional filter trigger.

Design System Alignment

This component is built by composing the atomic 'TextInput' and 'Button' components. It utilizes the design system's color, spacing, border, and typography tokens to ensure visual consistency with the rest of the platform.

Responsive Specifications

Mobile Adaptations

  • The search bar takes up the full available width.
  • The submit button collapses to an icon-only button to save space.

Tablet Adaptations

  • The search bar maintains its desktop appearance but may have a wider max-width.

Desktop Enhancements

  • The search bar has a defined max-width and is typically centered in the header.
  • The submit button includes text for clarity.

Accessibility Features

Keyboard Support

The component is fully navigable via keyboard. 'Tab' moves between the input field and buttons. 'Enter' in the input field submits the form.
